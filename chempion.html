Highlight 01

Status as the first thing you see

Every chemical in the inventory carries a status — in stock, expiring soon, or out of stock. Color-coded pills make urgency readable before you process a single number. Green, amber, red: the shelf-scan logic, digitized.

Most inventory checks are not "show me everything." They're "is anything wrong?" The dashboard answers that in under a second — then lets you drill down.

Highlight 02

Actions in the interface, not in the code

Adding a chemical used to mean editing a Python file. Now it's a form. The entire workflow — add, search, filter, export — happens through the UI. Lab staff can manage inventory independently.

Highlight 03

Four numbers that tell the whole story

The dashboard opens on four KPI cards — total chemicals, total value, expiring soon, low stock. These four numbers answer the question every lab manager actually asks when they open an inventory system.

Data density is not the goal. The goal is the fastest path from "I opened the app" to "I know what needs my attention." Four cards get you there.

The Problem

A tool built for one, used by all.

ChemPion started as a Jupyter notebook — a developer's tool for tracking lab chemicals. Functional if you knew Python. Inaccessible if you didn't. The data was all there. The interface wasn't designed for the people who needed it most.

01
No overview. Checking stock meant running a cell and reading raw output. Critical alerts — low stock, expiry — were buried in code.
02
No visual hierarchy. A chemical running out looked identical to one with years of shelf life. Status was invisible.
03
No actions. Adding, updating, exporting — all required editing code directly. Non-developers couldn't use it at all.

Result

From notebook to dashboard.

The same data, redesigned for the entire lab. Five fully functional sections, real-time search and filtering, form-based workflows for all common actions, and one-click CSV export.
